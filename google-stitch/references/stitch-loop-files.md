# Stitch Loop Files

## Suggested Structure

```text
.stitch/
├── DESIGN.md
├── SITE.md
├── next-prompt.md
├── metadata.json
└── designs/
```

## `SITE.md`
Document:
- site/app vision
- Stitch project title/id
- existing pages
- roadmap
- creative backlog

## `next-prompt.md`
Use for the next screen/page objective with:
- YAML `page:` frontmatter
- a clear page purpose
- a compact DESIGN SYSTEM block
- a simple page structure list

## `metadata.json`
Persist:
- project id
- screen ids
- source screen references
- device type
- design theme snapshot
